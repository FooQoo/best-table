# 計画

## 現在の目的

検索結果の地図上で、表示中の店舗群をコンテキストに AI へ相談できるチャット体験を追加する。

現状は `/results` で店舗一覧・地図・右側詳細パネルを見ながら候補を選べるが、「この条件ならどれを優先すべきか」「役員接待なら避けるべき候補はどれか」「比較に入れるならどの組み合わせがよいか」といった横断的な相談は、ユーザーがカードや比較表を自力で読み解く必要がある。

このサイクルでは、地図右下に AI チャットボタンを置き、押すと地図エリア内の右側から AI チャットパネルが出るようにする。チャットは現在 `/results` に読み込まれている店舗すべてをコンテキストにし、地図上の候補を見ながら相談できる位置づけにする。

あわせて、単一店舗の詳細ページと単一店舗向け質問 API は廃止する。店舗ごとの詳細は `/results` の右側詳細パネルに集約し、AI 相談は地図コンテキストのチャットに一本化する。

## 実装範囲

画面・パスの実装対象:

- MAP 付き検索結果: `/results`
- 地図右下の AI チャットボタン: `/results`
- 地図エリア内の右スライド AI チャットパネル: `/results`
- 表示中店舗群をコンテキストにした AI 相談 resource route
- 単一店舗詳細ページ `/stores/:storeId` の廃止
- 単一店舗質問 API `/api/stores/:storeId/ask` の廃止

実装対象外:

- 実予約
- 実在庫連携
- 地図上での空席表示
- AI による新規店舗検索や地図範囲検索
- チャット履歴の永続化
- 複数ユーザー共有、同席者への共有リンク
- 店舗詳細ページの別 URL としての再実装

## プロダクトフロー計画

1. ヒアリング入力: `/hearing`
2. 検索結果を取得し、左に店舗一覧、右に地図を表示: `/results`
3. 地図右下の AI チャットボタンを押す
4. 地図エリア内の右側から AI チャットパネルが出る
5. 初回画面には、地図上の店舗群に対するよくある質問を表示する
6. ユーザーが質問する
7. AI は現在 `/results` に読み込まれている店舗群とヒアリング条件だけを根拠に回答する
8. 回答後、次のおすすめ質問を4つ提示する
9. ユーザーは回答を見ながら店舗カードの選択、比較追加、比較画面への遷移を続けられる

## マイルストーン1: 地図内 AI チャット UI

目的:

地図を見ながら相談できる位置に AI チャット導線を置き、既存の店舗一覧・地図操作・比較導線を壊さずに開閉できるようにする。

タスク:

- `/results` の地図右下に AI チャットボタンを追加する。
- ボタンを押すと、地図エリア内の右側から AI チャットパネルが出るようにする。
- パネルは地図エリア内に重ね、左の店舗一覧は覆わない。
- パネルを開いても、店舗一覧の閲覧、比較追加、比較画面への遷移は維持する。
- パネル内の初回画面に、よくある質問を表示する。
- チャットパネルの閉じる操作を用意する。
- 地図読み込み失敗時でも、チャットボタンとパネルは店舗リストのコンテキストで使える状態にする。

受け入れ条件:

- `/results` の地図右下に AI チャットボタンが表示される。
- ボタンを押すと、地図エリア内の右側から AI チャットパネルが出る。
- パネルは店舗一覧を覆わない。
- 初回画面に、地図上の店舗群に関するよくある質問が表示される。
- パネルを閉じると地図表示へ戻れる。
- チャットパネルを開いた状態でも、店舗カードの比較追加と比較画面への遷移が使える。

## マイルストーン2: 地図コンテキスト AI 相談

目的:

表示中店舗群とヒアリング条件を根拠に、候補選びを前に進める短い AI 回答を返せるようにする。

タスク:

- 地図コンテキスト相談用の resource route を追加する。
- リクエストには、現在 `/results` に読み込まれている `Restaurant[]`、ヒアリング条件、質問文を含める。
- サーバー側で `Restaurant[]` の形状と質問文の長さを検証する。
- AI には、表示中店舗群とヒアリング条件だけを根拠に回答させる。
- 空席、予約成立、未取得の口コミ・メニュー本文などを断定しないプロンプトにする。
- 回答は短く、比較・懸念・次アクションが分かる形にする。
- AI 失敗時は、店舗一覧・地図・比較操作を維持したままエラーを表示する。

受け入れ条件:

- よくある質問または自由入力から質問できる。
- AI 回答が表示中店舗群に含まれる店舗名を使って相談に答える。
- 回答は空席や予約成立を断定しない。
- 回答が失敗しても、既存の検索結果と比較操作は消えない。
- 質問文が空、または長すぎる場合は送信できない。

## マイルストーン3: 次のおすすめ質問

目的:

1回の回答で終わらず、次に確認すべき観点を4つ提示して、比較判断を深められるようにする。

タスク:

- 初回表示用のよくある質問を定義する。
- 回答後に表示する「次のおすすめ質問」を4つ生成または選択する。
- おすすめ質問は、表示中店舗群、相手種別、重視条件、直前の質問・回答に合わせる。
- おすすめ質問を押すと、そのまま次の質問として送信できる。
- 質問候補は予約確定や空席断定を誘導しない文言にする。

受け入れ条件:

- 初回画面では、質問前のよくある質問が表示される。
- 1回質問すると、回答の下に次のおすすめ質問が4つ表示される。
- おすすめ質問を押すと、次の質問として送信される。
- おすすめ質問は「どの店を比較に入れるべきか」「懸念が大きい候補はどれか」など、地図上の店舗群を前提にした内容になっている。

## マイルストーン4: 単一店舗詳細ページ・質問 API の廃止

目的:

店舗詳細を別 URL と単一店舗 Q&A に分散させず、検索結果の詳細パネルと地図コンテキスト AI チャットに集約する。

タスク:

- `/stores/:storeId` ルートを削除する。
- `/api/stores/:storeId/ask` resource route を削除する。
- `StoreDetailScreen` と `StoreAskPanel` を削除または未使用化する。
- 店舗カードや比較画面から `/stores/:storeId` へ遷移する導線が残っていないことを確認する。
- 仕様書の「店舗詳細 AI」を「地図コンテキスト AI チャット」へ更新する。
- 単一店舗の詳細情報は `/results` の右側詳細パネルで確認する方針に統一する。

受け入れ条件:

- `/stores/:storeId` がアプリの正式ルート一覧から消える。
- `/api/stores/:storeId/ask` が resource route 一覧から消える。
- 検索結果の店舗カードクリックは `/results` 内の詳細パネル表示に留まる。
- AI 相談は `/results` の地図チャットに集約される。
- `pnpm run typecheck` と `pnpm build` が通る。

## 成功指標

ユーザー指標:

- 検索結果画面で、比較候補を選ぶ前の相談ができる。
- 地図上の候補を見ながら、用途に合う候補・避けたい候補を把握できる。
- 比較に追加する店舗を選ぶ時間が短くなる。
- 比較候補の選定理由を説明しやすくなる。
- 店舗詳細ページへ遷移せず、検索結果画面内で判断材料を得られる。

開発指標:

- 地図チャット UI が `/results` の店舗一覧・地図・比較トレイと競合しない。
- AI 相談 API は表示中 `Restaurant[]` とヒアリング条件だけを入力にし、サーバー側で検証される。
- 単一店舗詳細ページと単一店舗質問 API が削除され、ルート構成が単純になる。
- `pnpm run typecheck` と `pnpm build` が通る。

## モック UI 方針

このサイクルは新しいユーザー可視 UI（地図右下の AI チャットボタン、右スライドパネル、FAQ、次のおすすめ質問）を含むため、`UNIT_OF_WORK.md` を生成する前に軽量な mock UI を実装して確認する。

mock UI は本実装の足場として残す想定で、まずは `/results` の地図エリア内にチャットボタンとパネルを追加する。初回 FAQ と回答後のおすすめ質問4件は固定文言で用意し、AI API 接続は Phase 5 の UoW 分解後に実装する。mock UI 確認後、仕様に合わせて `UNIT_OF_WORK.md` を生成する。

## モック UI レビュー結果

2026-07-05 に mock UI を確認し、このデザインで進める方針を決定した。

採用した方針:

- チャットボタンは地図右下に置く。
- チャットパネルは地図エリア内の右側からスライド表示する。
- 店舗詳細パネルを開いた状態で AI チャットボタンやチャットパネルを操作しても、店舗詳細パネルは閉じない。
- チャット入力は初期1行表示とし、長文入力や `Shift+Enter` の改行に合わせて縦方向に自動拡張する。
- 通常の `Enter` は送信、`Shift+Enter` は改行とする。
