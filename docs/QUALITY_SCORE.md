# 品質スコア

## 目的

Best Table には、一般的なレストラン人気度ではなく、接待・会食での失敗しにくさを表す判断スコアが必要です。

主なスコアが答えるべき問い:

> この会食で失敗しにくい候補か？

## 主要スコア

主要ラベル:

- 接待安全度

これは、現在の会食文脈に対する総合的な安心度スコアです。

影響する要素:

- 相手種別
- 選択された重視条件
- 予算レンジ
- 席・個室要件
- 口コミや根拠データ
- アクセスや到着時の不安

## 基本評価軸

- 会話しやすさ
- 個室・席の安心感
- 接客の安定感
- 格式感
- アクセスのわかりやすさ
- 予算適合度

## 文脈ごとの重み付け

### 重要顧客・役員クラス

強調する要素:

- 個室・席の安心感
- 接客の安定感
- 格式感
- 懸念の深刻度

許容度を下げる要素:

- 個室がない
- 騒がしい雰囲気
- 接客に関する根拠が弱い
- アクセスが分かりにくい

### 初回の取引先・商談前後

強調する要素:

- 会話しやすさ
- 堅すぎない雰囲気
- アクセスの分かりやすさ
- 接客の安定感

許容度を下げる要素:

- 騒がしい、または席が狭い
- カジュアルすぎる、または堅すぎる
- 場所が分かりにくい

### 社内上司・幹部

強調する要素:

- 予算適合度
- 落ち着き
- 使いやすさ
- アクセス

許容度を下げる要素:

- 予算から大きく外れる
- カジュアルすぎる
- 席の適性が不明確

### お礼・懇親

強調する要素:

- 特別感
- 居心地
- もてなし感
- 印象に残る雰囲気

許容度を下げる要素:

- 体験が汎用的または事務的すぎる
- 接客面の懸念がある
- 会話しづらい席

### 関係を深めたい相手

強調する要素:

- 話しやすさ
- 印象に残るが、肩肘張らない雰囲気
- 居心地
- アクセス

許容度を下げる要素:

- 堅すぎる
- 騒がしい
- 会話しづらい席

## 重視条件による調整

選択された重視条件に応じて、対応する評価軸の重みを上げる。

- 落ち着いて話せる -> 会話しやすさ、騒がしさ、席間隔
- 個室・半個室を優先 -> 個室・席の安心感
- 失礼のない格式感 -> 格式感、相手への適切さ
- 接客の安定感 -> 口コミから見た接客安定度
- 駅から迷わず行ける -> アクセスのわかりやすさ
- 予算内に収めたい -> 予算適合度
- 苦手・避けたい条件がある -> 自由入力に基づく減点ルール

重視条件は最大3つまでに保ち、スコアの意味を解釈しやすくする。

## 根拠入力

有用な情報カテゴリ:

- 店舗基本情報
- 席・個室の説明
- コース・メニュー説明
- 店舗紹介文
- 口コミ本文
- 口コミ評価
- 写真カテゴリと見える雰囲気
- アクセス説明
- 空席情報と営業時間

## 派生フィールド

本番を想定した派生データ:

- 接待安全度
- 会話しやすさスコア
- 個室安心度
- 接客安定度
- 格式感スコア
- アクセス安心度
- 予算適合度
- 懸念点タグ
- 推奨利用シーン
- 根拠要約
- 信頼度
- 生成日時

## スコアリング方針

スコアを、根拠のない保証のように扱ってはいけない。

ルール:

- 根拠が不足している場合は保守的に評価する。
- 高スコアの店舗でも懸念点を表示する。
- 確認できた強みと推測上の強みを区別する。
- 個室の空き状況や在庫を捏造しない。
- 総合スコアが高くても重大な懸念を隠さない。
- 数値よりも理由を重視する。

## コメント例

> 個室の記載があり、口コミでも接客の丁寧さに言及が多い。重要顧客との会食では安心度が高い一方、駅から少し歩く点は事前案内したい。

このコメントが有効な理由:

- 肯定的な根拠がある。
- 会食文脈に合わせた適性が示されている。
- 具体的な懸念がある。
- 実行しやすい次アクションがある。
